---
description: 指定プロジェクトについて全体の成果をまとめ、ダッシュボードの「成果」タブに保存する
---

`$ARGUMENTS`（プロジェクトのパス、リポジトリ名、または説明）について、**そのプロジェクト全体で
何をやったか**を調べて成果サマリを Markdown でまとめ、ローカルのダッシュボードに保存してください。

毎日の細かい修正の羅列ではなく、**プロジェクト単位の成果**としてまとめるのが目的です
（職務経歴・評価・ポートフォリオに転用できる粒度）。**Anthropic API キーは不要**（このセッションが書きます）。

## 手順

1. プロジェクトを把握する。`$ARGUMENTS` がパスならそのコードを探索し、必要なら
   `~/.claude/projects` 配下の該当プロジェクトの作業ログ（jsonl）や git log も参照して、
   **全体で何を作り・何を解決し・どんな結果が出たか**を理解する。
   不明点は推測で埋めず、わかる範囲でまとめる。

2. 本文を以下の構成で作成する（日本語・Markdown）:
   ```
   # プロジェクト名
   ## 概要
   何のプロジェクトか・自分の関わりを2〜4行で。
   ## やったこと
   プロジェクト全体で担当・実装した主要な内容（箇条書き）。
   ## 成果・インパクト
   結果や数値（例: 工数◯%削減 / CV◯件増 / 公開・リリース）。数値が無ければ定性的成果でよい。
   ## 使った技術・役割
   技術スタックと自分のロール。
   ```

3. メタ情報を決める:
   - `project`: プロジェクト名（一覧の見出しになる）
   - `period`: 期間（例 `2026 Q1–Q2` や `2026-01〜2026-06`）。不明なら空文字 `""`。

4. ダッシュボード API に保存する。`npm run dev` 起動中なら:
   ```bash
   curl -sS -X POST http://localhost:3000/api/achievements \
     -H "Content-Type: application/json" \
     -d "$(jq -nc --arg project "$PROJECT" --arg period "$PERIOD" --arg body "$BODY" \
            '{project:$project, period:$period, body:$body}')"
   ```
   （`$PROJECT` `$PERIOD` `$BODY` に上で作った値を入れる）

5. 保存したプロジェクト名を表示する。「成果」タブに反映される。

## サーバーが起動していない場合（フォールバック）

`data/achievements.json` を直接編集する（無ければ `[]` から作る）。配列に次の形で 1 件追加して保存:
```json
{
  "id": "<現在時刻のミリ秒など一意な文字列>",
  "project": "プロジェクト名",
  "period": "期間 or 空文字",
  "body": "上で作った Markdown 本文",
  "createdAt": "<ISO8601 の現在時刻>"
}
```
（このファイルは `.gitignore` 済みなので、公開リポジトリには含まれない。）
